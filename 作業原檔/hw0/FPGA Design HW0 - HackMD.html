FPGA Design HW0 - Verilog Code Practice

繳交時間

2025/03/10(一) 晚上19:10，逾時拒收並以0分計算

評分標準

60% 模擬結果正確性
40% 文件說明完整性

禁止抄襲

作業繳交

資料夾格式

  • /FPGA_HWX_GroupX
    • /src
      • SET.v // 自己建
      • SET_tb.v // moodle 會提供
    • 說明文件(.md或.pdf)

說明文件

  1. 內容至少要有組別,所有組員學號,電路設計說明,模擬結果截圖並分析說明
  2. 如果是 Word 或是 PPT 請轉成pdf繳交

上傳規範

  1. 請將最後的資料夾壓縮成.zip
  2. 每一組派其中一位同學繳交一份至moodle即可

題目(moodle上有放)

2015 IC Design Contest Preliminary 研究所類標準元件數位電路設計 - 指定集合內覆蓋之元素個數計數器

問題描述

本題目須完成利用多個圓形之集合運算計算出圖內覆蓋的頂點個數(Counting the number of vertex covers by set-theoretic operations of multiple circle graph)。此系統須將測試模組所提供在二維平面空間 8x8 座標系統。

如下圖所示，在二維平面空間 8x8 座標系統內共有 3 個圓形。input [23:0] central & input [11:0] radius 會負責提供我們這三個圓形的圓心和半徑的資訊(其中圓心座標與半徑皆是正整數)，以供我們針對這三個圓涵蓋的所有點(包含圓周內與剛好坐落在圓周上的點)所形成的集合(A, B, C)進行各種集合運算，電路最後輸出指定集合內的元素個數。

Code 說明

本次作業請以以下的 code 做為基礎進行













module SET (clk ,rst, en,central,radius, mode, busy, valid, candidate);
    input clk;
    input rst;
    input en;
    input [23:0] central; // 三個圓心座標的拼接值
    input [11:0] radius; // 三個圓半徑的拼接值
    input [1:0] mode; // 表示選擇哪種模式
    output busy; // 表示模組忙碌狀態
    output valid; // 表示輸出的值是否有效
    output [7:0] candidate; // 所求

endmodule

<注意>

  1. top module名稱(SET)及I/O大小寫請勿更動
  2. 如有引入其他模組、檔案，請在說明文件當中明確地說明其用意

訊號說明如下

Signal Name Direction Width(bit) Description
rst input 1 非同步系統重置訊號。當此訊號為 1 時表示系統重置。
clk input 1 系統時脈訊號。
en input 1 資料有效信號。當此訊號為 1 時表示輸入資料為有效。
central input 24 集合座標資料。其組成為{x1,y1,x2,y2,x3,y3}，i.e. central[23:20], central[19:16], central[15:12], central[11:8],central[7:4], central[3:0]分別為集合A的X軸座標(x1), 集合A的Y軸座標(y1), 集合B的X軸座標(x2), 集合B的Y軸座標(y2), 集合C的X軸座標(x3), 集合C的Y軸座標(y3)
radius input 12 集合半徑資料。其組成為{r1,r2,r3}，i.e. radius[11:8], radius[7:4], radius[3:0]分別為集合 A 的半徑值 r1, 集合 B 的半徑值 r2, 集合 C 的半徑值 r3
mode input 2 mode 為指示進行運算模式信號。
busy output 1 系統忙碌指示訊號。當此訊號為 1 時表示系統為忙碌中。
valid output 1 valid 為有效的資料輸出指示信號。當信號 valid 為高位準時，於 candidate 輸出埠所輸出之資料為有效資料。
candidate output 8 輸出題目所指定的元素個數。
更詳細的訊號控制描述

文字搭配圖片服用更佳

當 reset 結束後。每當測試模組偵測到 busy = 0 且經時脈訊號負緣觸發時會立刻送出輸入致能訊號 en、集合座標資料 central、集合半徑資料 radius 及運算模式指示訊號 mode;而當 en = 1 且經時脈訊號正緣觸發之 central、radius 及 mode 為有效輸入訊號。
系統需在接收到有效輸入訊號後將 busy 拉為 1 表示系統忙碌中，並利用運算模式指示訊號mode 所指示之計算出其集合空間裡所涵蓋的元素個數。接著系統須將 valid 拉為 1 並同時將前述計算完成之集合座標點數目利用 candidate 訊號輸出。接著請再次將 busy 設定為 0 表示系統閒置，測試模組將在偵測到 busy = 0 且經時脈訊號負緣觸發後輸出下一筆待測訊號。

Mode 說明

集合運算模示訊號 mode 為 2 bits，共有四種模式，系統須依訊號指示擷取有效資料進行運算。

  1. 集合模式指示訊號(mode)訊號為 2’b00：
    A
    說明：計算一個 A 集合在二維 8x8 空間裡所涵蓋的座標元素個數，其中 central 信號 (x1,y1)，radius 信號 r1 為有效資料。
    圖一、A 集合(所涵蓋元素個數為29)
  2. 集合運算模式指示訊號(mode)訊號為 2’b01：進行
    (A∩B)     運算
    說明：計算 A、B 兩個集合，2 維 8x8 座標空間裡，交集所涵蓋的元素座標點個數，其中 central 信號(x1,y1)、 (x2,y2)及 radius 信號 r1、r2 為有效資料。
    圖二、
    A∩B     運算(所涵蓋元素個數為13)
  3. 集合模式指示訊號(mode)訊號為 2’b10：進行
    (A∪B)−(A∩B)     運算
    說明：計算 A、B 兩個集合在 2 維 8x8 座標空間裡，差集所涵蓋的元素座標點個數，其中 central 信號 (x1,y1)、(x2,y2)及 radius 信號 r1、r2 為有效資料。
    圖三、
    (A∪B)−(A∩B)     運算(所涵蓋元素個數為30)
  4. 集合模式指示訊號(mode)訊號為 2’b11：
    (A∩B)+(B∩C)+(A∩C)−(A∩B∩C)     運算
    說明：計算 A、B、C 三個集合在 2 維 8x8 座標空間裡，每個兩個集合作交集元素座標點個數，並排除三 個集合所交集之元素座標點個數，其中 central 信號(x1,y1)、(x2,y2)、(x3,y3)及 radius 信號 r1、r2、 r3 為有效資料。
    圖四、
    (A∩B)+(B∩C)+(A∩C)−(A∩B∩C)     運算(所涵蓋元素個數為14)
