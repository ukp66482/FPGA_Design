FPGA Design HW1

繳交時間

2025/03/24(一) 晚上19:10，逾時拒收並以0分計算

評分標準

60% 實作結果正確性
40% 文件說明完整性

禁止抄襲

Outline

  • Problem 1 - RGB LED(30%)
  • Problem 2 - Traffic Light(60%)
  • Problem 3 - Q & A(10%)

作業繳交

資料夾格式

  • /FPGA_HWX_GroupX
    • /Problem1
      • /src
        • *.v // 要幾個.v自己建
      • /xdc
        • *.xdc
      • /bit
        • *.bit
    • /Problem2

<注意>
不用上傳整個Vivado Project，只需提供source file(.v)、xdc(.xdc)、bitstream(.bit)即可

說明文件

  1. 內容至少要有組別,所有組員學號,電路設計說明,Problem3的回答,YouTube網址
  2. 如果是 Word 或是 PPT 請轉成pdf繳交

YouTube標題：FPGA_HWX_GroupX_ProblemX

影片中需呈現其功能完整性、正確性
並以後製的方式或是在說明文件中標註其功能的Timecode
或是邊錄影邊解說(較佳)

<注意>
某些手機、平板進行錄影時會看到燈光閃爍，請隊友們互相協助，盡量避免此情況

上傳規範

  1. 請將最後的資料夾壓縮成.zip
  2. 每一組派其中一位同學繳交一份至moodle即可

Problem 1 - RGB LED(30%)

實作一個電路，可依據不同的 switch 輸入改變 RGB LED

  • Switch 負責切換 RGB LED 燈號
  • 1 個 RGB LED 的顏色由 3 個 bit 控制：分別從左至右依序為 RGB

Spec

Switch RGB LED led4_rgb
2'b00 White 3'b111
2'b01 Red 3'b100
2'b10 Green 3'b010
2'b11 Yellow 3'b110

Problem 2 - Traffic Light(60%)

實作一個可調節時間的紅綠燈

  1. 使用左邊兩個 RGB LED 實現一個路口不同方向的兩個號誌燈

  2. 透過左邊兩個開關決定現在的功能為何、燈號為何

    如果 State = 正常運作(00)

    兩個號誌燈分別為T1, T2，R代表紅燈、Y代表黃燈、G代表綠燈

    • 每次重新切換到這個 State 請以 T1G T2R 狀態開始運作
    • Default時長：一紅一綠亮5秒，一紅一黃亮1秒，兩紅亮2秒
      • 1秒不必是精確的1秒，可以直接使用LAB提供的除頻器(divider)

    如果 State = 調整時長(01,10,11)

    使用右邊的 button 們根據 switch 調整各個燈號的時間

    各時長最短為1秒，最長為15秒

    • 正常運作的模式下需用右邊的 4-bit LED 顯示該燈號當前剩餘秒數(in binary)
    • 調整時長的模式下需用右邊的 4-bit LED 顯示該燈號之設定秒數(in binary)

  4. 到目前為止我們可以發現到BTN[3]沒用到，其實他是我們系統的Global reset鍵。只要按下這個鍵的瞬間，就讓左邊的兩個RGB LED亮白光、右邊的4-bit LED暗掉、各個燈號的時長回到預設值、如果 Switch=00 則回到 T1G T2R 狀態。

  5. 最後實作完畢後請看下圖確認功能是否正確

Problem 3 - Q & A(10%)

  1. 為什麼要加入 blinky. xdc？

    Hint：https://docs.xilinx.com/r/en-US/ug903-vivado-using-constraints/Constraints-and-Object-Queries 的 Chapter 3

  2. Vivado 的開發流程中 Synthesis 和 Implementation 的結果差異在哪？

    Hint：觀察兩個步驟的Device圖
